Module 2: Product Design

Generate Your AI-Assisted Persona

Exercise 8 minutes

Use AI to synthesize user feedback into a persona, then validate and refine it.

Copy this prompt → ChatGPT/Claude:

I'm a product manager building [your product type]. Here's raw user feedback from [source]: "[Paste 2-3 real quotes or summarize findings]" Based on this feedback, generate a user persona: 1. Name and brief background (1 sentence) 2. Primary goal when using products like mine 3. Top 3 frustrations/pain points 4. One unexpected insight about their behavior 5. A quote that captures their mindset Format as a concise persona card.

Your validation checklist:

☐ Does this feel like a REAL person I've met?

☐ What context is missing that I know?

☐ What assumption should I challenge?

☐ What emotional depth is underdeveloped?

Refinement prompts (use at 5:00):

If too generic: "Add a concrete scenario where they get frustrated"

If emotion missing: "What do they FEEL when hitting this pain point?"

To challenge: "You assumed X. In my experience, users often Y. Revise."

Module 2: Product Design

Map the Journey with AI Assistance

Exercise 7 minutes

Use the 5 E's framework to map your persona's complete journey, then add your human insight.

Copy this prompt (use your persona):

Using [persona name] from before, map their journey through [your product] across 5 stages: 1. ENTICE: How do they first hear about solutions? 2. ENTER: What's their first interaction? 3. ENGAGE: What happens during core usage? 4. EXIT: How does the experience end? 5. EXTEND: What brings them back (or not)? For each stage include: - What they DO (action) - What they FEEL (emotion) - What could go WRONG (pain point) - What would make it GREAT (opportunity)

The 5 E's Framework:

ENTICE

"I heard..."

→

ENTER

"Let me try..."

→

ENGAGE

"I'm using..."

→

EXIT

"Done..."

→

EXTEND

"Back?"

Your human additions:

1. Find where AI got emotion wrong:

"At stage ___, they actually feel ___"

2. Mark highest-stakes moment:

This is your design priority 🎯

Module 3: Your One-Page PRD

Your PRD is your prompt

Guest Speaker from ATB Financial

How regulated industries approach generative AI for product development

GenAI in regulated environments:

  • Risk-sensitive approach — balancing innovation with compliance
  • Guardrails matter — responsible AI use in banking
  • Human-in-the-loop — AI augments, doesn't replace judgment
  • Audit trails — documenting AI-assisted decisions

Why a PRD matters even more now:

Better prompts: Structured input → dramatically better AI output

Traceability: Connect every feature to a real user need

Scope control: Define what's out so AI doesn't hallucinate features

Your One-Page PRD structure:

Problem + Persona

Who you're building for and why (from Modules 1 & 2)

User Stories

"As a [user], I want [action] so that [benefit]"

Acceptance Criteria

Testable definition of done for each story

Out of Scope

What you're NOT building — keeps AI focused

Success Metrics

How you'll know it's working for real users

Module 3: Your One-Page PRD

Live Demo: Journey Map → One-Page PRD

PRD generation prompt:

I'm building a [product] for [user persona from Module 2] who struggles with [pain point]. Here's my user journey: - Trigger: [what causes them to need this] - Step 1: [action] → Pain: [frustration] - Step 2: [action] → Pain: [frustration] - Success: [how they know it worked] Generate a one-page PRD with: 1. Problem statement (2-3 sentences) 2. Target persona (1 sentence summary) 3. 3-5 User Stories: "As a [user], I want [action] so that [benefit]" 4. Acceptance criteria for each story 5. Out of scope for MVP 6. One success metric I can measure in week 1

Key insight: This PRD becomes your build prompt in Module 4. The better your PRD, the better your AI-built MVP.

Example PRD output:

Problem

Indie hackers lose valuable user feedback because it arrives across scattered channels with no fast way to capture it.

User Story 1:

As an indie hacker, I want to paste feedback from any source so that I don't lose valuable user insights.

Acceptance Criteria:

  • Single text input accepts any format
  • Submit takes < 2 clicks
  • Confirmation shown on success

Success Metric:

3+ feedback entries captured in first 7 days of use

Your PRD maps directly to your build prompt:

  • Problem + persona → context the AI needs
  • User stories → feature list
  • Acceptance criteria → specific behaviors
  • Out of scope → what to exclude
  • Success metric → what to measure

Tools: v0.dev, Bolt.new, Replit, Claude, Cursor, GitHub Copilot, etc.

PRD-informed build prompt:

Build this MVP based on my PRD: PROBLEM: [paste your problem statement] PERSONA: [paste your persona summary] USER STORIES: [paste your user stories] ACCEPTANCE CRITERIA: [paste key acceptance criteria] NOT IN SCOPE: [paste out of scope items] Style: Dark mode, minimal, fast-loading.
Module 4: Build Sprint

Build & Deploy Your MVP

Build Sprint 15 minutes

Turn your PRD into a deployed product

Steps:

  1. Open v0.dev or Bolt.new

    Paste your requirements as the prompt

  2. Iterate once

    Refine based on output

  3. Deploy

    Get your live URL

Deployment options:

  • v0: Click Deploy → Vercel
  • Bolt: Already hosted, click Share
  • Replit: Click Run → copy URL

Keep it simple:

  • Use localStorage (no backend needed)
  • Focus on your #1 user story first
  • Working > perfect
  • You can iterate after launch

If stuck:

Paste your requirements + code into Claude and ask:

"Help me implement [specific user story]. Here's my current code..."

Module 5: GTM Sprint

Step 2: Build Your Lead List in Clay

GTM Sprint 8 minutes

Create a table with 10-15 qualified leads

  1. Create a new table in Clay

    Name it after your product/ICP

  2. Add people using "Find People"

    Use LinkedIn search or import from a list. Filter by your ICP criteria.

  3. Enrich with company data

    Add columns for company size, industry, recent news

  4. Find email addresses

    Use Clay's email finder enrichment

Goal: 10-15 people who match your ICP with valid emails

Module 5: GTM Sprint

Step 3: Generate Personalized Outreach

GTM Sprint 8 minutes

Use Clay's AI to write personalized messages for each lead

Add an AI column:

  1. Click "Add Column" → "AI"
  2. Choose "Generate Text"
  3. Write your prompt template
  4. Reference other columns with {{}}
  5. Run for all rows

Outreach prompt template:

Write a short, friendly cold email to {{First Name}}. Context: - They are {{Title}} at {{Company}} - Company does: {{Company Description}} - I built: [your MVP description] - My MVP URL: [your deployed URL] The email should: - Be under 100 words - Reference something specific about them - Ask if they'd try the product - No salesy language
